# เปลี่ยนจาก CLI เป็นเว็บแอป

เทมเพลตต้นทาง (booking-template) เป็น CLI ล้วน แต่กลุ่มผู้ใช้จริงคือลูกค้าทั่วไปที่ต้องจองเองและเจ้าของร้านที่ไม่ถนัด terminal เราจึงสร้างเป็น **เว็บแอป**: หน้า public ให้ลูกค้าจอง และหน้า `/admin` ให้ร้านจัดการ โดยยัง reuse ตรรกะหลักของเทมเพลต (กฎเวลาทับซ้อน, การเก็บข้อมูล JSON, สถานะ confirmed/cancelled) เป็น backend

## Consequences

- ต้องมี HTTP server + หน้าเว็บ เพิ่มจาก CLI เดิม
- ย้อนยาก: โครงสร้างทั้งโปรเจกต์ (API, หน้าเว็บ, การ deploy) ออกแบบรอบแกนเว็บ ถ้าจะกลับไป CLI ต้องรื้อ
